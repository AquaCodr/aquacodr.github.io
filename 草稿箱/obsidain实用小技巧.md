在 Obsidian 中，你提到的 `> [!info]` 这种语法被称为 **Callouts（引注框）**。它是基于微软文档风格的一种扩展语法，非常适合用来美化笔记、突出重点。

以下是一份精心整理的 Obsidian 常用语法手册，涵盖了 Callouts 全家桶以及一些进阶的排版技巧。

---

## 1. Callouts（引注框）语法大全

基础格式如下：

Markdown

```
> [!type] 标题（可选）
> 内容部分
```

### 常用内置类型表

|**类型名称**|**语法标识**|**视觉效果描述**|
|---|---|---|
|**信息**|`[!info]` / `[!note]`|蓝色，适合记录补充说明|
|**摘要**|`[!abstract]` / `[!summary]`|青色，适合写章节概括|
|**完成**|`[!success]` / `[!check]` / `[!done]`|绿色，适合打卡、任务完成|
|**提示**|`[!tip]` / `[!hint]`|青色/绿色，适合记录小技巧|
|**警告**|`[!warning]` / `[!caution]` / `[!attention]`|黄色，提醒注意潜在问题|
|**危险**|`[!danger]` / `[!error]`|红色，警示严重错误|
|**引用**|`[!quote]` / `[!cite]`|灰色，书摘或名言专用|
|**疑问**|`[!question]` / `[!help]`|橙色，记录未解决的问题|

### 进阶技巧：可折叠框

如果你希望笔记页面更整洁，可以给引注框加上折叠功能：

- **默认展开：** `> [!info]+ 标题`
    
- **默认折叠：** `> [!info]- 标题`

## 2. 文本高亮与标注

除了基础的加粗和倾斜，Obsidian 还有几个非常有用的原生语法：

- **重点高亮：** `==这里会被涂上荧光笔==` （双等号）
    
- **行内代码：** `C语言函数`（反引号）
    
- **删除线：** `~~已经过时的信息~~`
    
- **双链引用：** `[[笔记标题]]` 或 `[[笔记标题#标题1|显示名称]]`
    

---

## 3. 任务列表（Task Lists）

在 Obsidian 中，任务列表不仅能看，还能直接点击勾选：

- [ ] 待完成任务：`- [ ] 任务内容`
    
- [x] 已完成任务：`- [x] 任务内容`
    
- [/] 正在进行：`- [/] 任务内容`（部分主题支持）
    
- [-] 已取消：`- [-] 任务内容`（部分主题支持）
    

---

## 4. 属性（Properties/YAML）

在笔记的最顶端输入 `---` 并回车，可以开启 **Properties** 界面，这对于管理你的“数字花园”非常有用：

YAML

```
---
tags: [学习/信号与系统, 编程/C语言]
status: In Progress
date: 2026-03-29
---
```

这样你就可以通过标签或 Dataview 插件快速筛选笔记。

## 6. 实用小插件推荐

如果你喜欢这种视觉增强，推荐在插件市场搜索：

1. **Admonition**：可以自定义更多颜色的 Callouts。
    
2. **Editing Toolbar**：像 Word 一样提供一个顶部工具栏，点一下就能插入这些语法。
    
3. **Style Settings**：如果你懂一点点 CSS，可以用它深度定制引注框的颜色。
